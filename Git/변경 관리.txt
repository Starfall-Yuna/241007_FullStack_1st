형상 관리 = 변경사항 관리 + 버전 관리

<변경 관리>
로컬저장소(각각의 컴퓨터) / 원격저장소(ex. 클라우드) 
각자 컴퓨터 - 깃허브(원격저장소)
git은 두 저장소를 중재

git init :: 
깃의 명령어를 사용할 준비함

(1) 다운로드 -> (2) 작업(수정) -> (3) 업로드
- 다운로드 :: pull(원격->로컬 당겨받기)
	- git pull [원격저장소_주소] [원격저장소_브랜치_이름]
- 업로드 :: 
	- git add [파일명] :: 변경사항을 추척할 파일 "추가"
	- git commit -m "[커밋메시지]" :: 변경사항 "기록"
	- git push [깃허브_리포지토리_주소] main ::
		업로드(로컬->원격 밀어넣기)
